# 一手资讯平台 — 想法探索

## 核心想法

当前互联网充满二手咨询：自媒体加工、AI流水线生成内容，浪费用户注意力。

**目标**：做一个只提供各官方渠道一手信息的平台，筛掉所有二手信息，时效性足够高。

---

## 信息层级定义

| 层级 | 示例 | 是否纳入 |
|------|------|----------|
| 0 级：原始事件 | 某机构发布的官方公告 PDF | ✅ |
| 1 级：直接引用 | 官方 Twitter/微博 原文 | ✅ |
| 1.5 级：官方摘要 | 新华社播发的政府公告 | 边界模糊 |
| 2 级：媒体报道 | 36氪/TechCrunch 的解读 | ❌ |
| 3 级：自媒体加工 | 公众号转述 + 观点 | ❌ |
| 4 级：AI 流水线 | 批量生成的资讯文章 | ❌ |

---

## 核心难点

### 1. 官方来源白名单（核心壁垒）
- 政府机构：相对清晰（gov.cn, nih.gov）
- 上市公司：SEC/EDGAR 的披露、官网新闻室
- 学术机构：arXiv、bioRxiv、DOI
- 科技公司：官方 Blog、GitHub Release Notes
- 组织/NGO：官网公告

白名单本身是产品的核心资产，需要持续人工维护。

### 2. 技术实现路径

```
数据采集层
├── RSS/Atom（大量官方机构提供）
├── 官方 API（Twitter API、政府 Open Data）
├── 定向爬虫（官网新闻室、IR 页面）
├── 订阅邮件解析
└── Webhook/Push（如 GitHub Releases）

过滤判断层（最难）
├── 域名白名单匹配
├── 发布主体验证（是否官方认证账号）
├── 内容指纹：是否是"原创发布"而非转载
└── AI 辅助检测：是否引用了其他来源

呈现层
├── 按来源分类（不做编辑加工）
├── 原文展示为主
├── 只标注：来源、发布时间、原始链接
└── 禁止算法推荐
```

### 3. 用户门槛问题
纯一手信息没有上下文，普通用户可能看不懂（专业文件）。

## 竞品调研（2026-03-24）

### 最接近的产品：[Primary News Source](https://primarynewssource.org/)

- 每天监控 **800+ 官方信息源**（美国政府机构、30+ 国际组织）
- 每周发布 **300-350 份**官方文件
- 数据库超过 **35,000 条**记录
- 核心理念："没有新闻报道，没有偏见，让文件自己说话"

**局限：**
- 主要聚焦美国政府文件，偏政府/政策领域
- 没有覆盖科技公司、学术、金融等领域
- 界面朴素，偏工具站
- 无中文/中国市场版本

### 其他相关产品

| 产品 | 定位 | 与目标的差距 |
|------|------|------------|
| **Ground News** | 对比同一事件不同媒体报道偏向 | 是二手信息的"分析"，不是过滤 |
| **Feedly / Inoreader** | RSS 聚合器 | 不区分一手/二手，靠用户自己配置 |
| **Google News** | 算法推荐新闻 | 完全是二手媒体聚合 |
| **SEC EDGAR** | 上市公司官方披露 | 垂直领域一手，只限财务 |
| **arXiv** | 学术预印本 | 学术领域一手，不做综合聚合 |

### 国内现状：**基本空白**

- 各部委有独立官网，但没有统一聚合入口
- 网信办有"互联网新闻信息稿源单位名单"（2025版），是官方媒体白名单，但非聚合平台
- 微信、微博等主流平台全是二手信息逻辑

## 机会判断

```
primarynewssource.org 证明了这个方向可行
但它只做了"政府文件"一个垂直，且只面向英文/美国市场

差异化机会：
├── 中文市场完全空白
├── 科技/AI/金融等商业领域无人做
└── 消费级产品体验（而非工具站）无人做
```

---

## 商业模式

- **To C 订阅制**：研究员、分析师、记者、投资人
- **To B API**：新闻机构、合规部门、舆情系统
- **To 公众免费基础版**：垂直领域先验证（科技、政策、医疗）
